# Web Audit — verdix.pages.dev (2026-07-21)

Metode: screenshot Playwright desktop 1440px + mobile 390px semua halaman (landing, /web, /web/agent/1, /web/create), scroll penuh, curl headers, checklist Vercel Web Interface Guidelines + skill seo-audit. Nol console error di semua halaman.

## Verdict singkat
Landing sudah kelas modern (dark, gradient, tx receipt live di hero, reveal animation, reduced-motion dihormati, alt lengkap, 1×h1, font self-hosted <55KB total, webp <52KB). Gap terbesar BUKAN landing — tapi halaman app `/web/*` yang beda dunia desainnya, campur bahasa di mode EN, dan metadata share yang bolong.

## P0 — paling ngaruh
1. **App `/web/*` tidak satu bahasa desain dengan landing.** Directory/Agent/Create pakai link biru default, tabel polos, header text-only. Landing = hackathon-winner, app = prototype. Juri klik "Launch App" dan kualitas persepsi jatuh di situ.
2. **Campur bahasa di mode EN** (`/web/agent/1`): "Komponen skor", "di BscScan", "di Membase", "terakhir", "Terpakai". Screenshot produk di landing (shot-profile/vault.webp) juga baked-in teks Indonesia ("Terpakai hari ini", "Hanya owner vault…"). Perlu sapu i18n + re-shoot screenshot versi EN.
3. **og:image & twitter:card TIDAK ADA** (landing cuma og:title/desc/type). Share di X/Discord/TG = teks polos tanpa card. Bikin 1200×630. Halaman app malah nol meta description/og semua.

## P1
4. **Mobile app**: nav header wrap 2 baris berantakan; tabel directory kepotong di 390px (kolom revenue) tanpa kontainer overflow-x → butuh scroll container atau layout kartu.
5. **API cold start**: `/api/agent/1` 6.4s cold, 0.11s warm. Leaderboard "live" di landing kerasa kosong/lama saat cold. Solusi: cron warm-ping atau cache edge.
6. **SEO plumbing**: canonical ✗, sitemap.xml 404, robots.txt cuma boilerplate content-signal CF (tanpa `Sitemap:`), JSON-LD ✗, custom 404 ✗.
7. **Security headers**: HSTS/X-Frame-Options/CSP semua kosong. Buat situs yang nyuruh connect wallet, XFO/CSP = proteksi clickjacking + trust signal. Gampang via file `_headers` Cloudflare Pages.

## P2 — biar setara web3 modern (Ethena/EigenLayer/Linea-tier)
8. Custom domain (verdix.xyz dsb) — `.pages.dev` keliatan proyek weekend di mata juri/VC.
9. Footer tanpa link X/Discord/Telegram (cuma GitHub/API/BscScan) — crypto tanpa jejak komunitas keliatan mati.
10. Logo bar "Built on/with" (BNB Chain, Unibase/Membase, ERC-8004) sebagai social proof — sekarang cuma teks.
11. Hero statis — web3 modern biasa ada subtle motion (gradient animasi/particle ringan); yang ada sekarang (tx receipts) sudah bagus, tinggal dipoles motion halus.
12. Dropdown EN pakai native select default, belum di-styling; skip-link belum ada.

## Yang sudah benar (jangan dirusak)
Hero copy tajam ("Reputation your AI agent can't fake" / "The chain said no. That's the product."), bukti on-chain klik-able, stats bar, tokenomics donut, roadmap, footer ERC-8004 positioning, mobile landing rapi, aria-label 8 titik, prefers-reduced-motion, theme-color.

Catatan artefak: screenshot full-page playwright nampak hitam karena reveal animation — di browsing asli konten normal.
